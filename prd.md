**A standalone Kubernetes controller built to demonstrate skills for contributing to [koordinator-sh/koordinator](https://github.com/koordinator-sh/koordinator)**

**Version:** 1.0  
**Repo Name (suggested):** `https://github.com/G26karthik/sandbox-warm-pool-controller' 
**Language:** Go 1.21+  
**License:** Apache 2.0  
**Purpose:** Portfolio project submitted alongside a Koordinator open-source contribution application, demonstrating direct familiarity with Koordinator's exact stack and the pre-warming mechanism described in their sandbox scheduling proposal.

---

## 1. Background & Motivation

Koordinator is a QoS-based Kubernetes scheduling system. One active area of development is **sandbox pod management** — running workloads inside hardware-isolated runtimes (gVisor, Kata Containers) for multi-tenant safety.

A core performance problem is **cold-start latency**: gVisor (`runsc`) and Kata (micro-VM boot) take 1–5 seconds to initialize, making on-demand sandbox creation unacceptable for latency-sensitive use cases (FaaS, AI inference, CI runners).

This project implements a **Warm Pool Controller**: a Kubernetes controller that pre-creates a configurable pool of idle sandbox pods so assignment requests are served in <100 ms by claiming a pre-warmed pod instead of booting a new one. This maps to the pre-warming mechanism in Koordinator's sandbox proposal and uses their exact technology choices.

---

## 2. Goals

1. Kubernetes controller in Go using `controller-runtime`.
2. CRD named `SandboxWarmPool` (group: `sandbox.koordinator.sh`, version: `v1alpha1`).
3. Controller maintains a pool of pre-initialized sandbox pods using gVisor or Kata `RuntimeClass`.
4. Track pods through a lifecycle state machine: `Pending → Idle → Assigned → Recycling → Terminating`.
5. HTTP API for consumers to claim and release warm pods atomically.
6. Prometheus metrics for pool depth, assignment latency, warm-up time.
7. Full unit tests using `envtest`.
8. Clean `README.md` that explains the Koordinator connection explicitly.

## 3. Non-Goals

- Not a PR into koordinator-sh/koordinator (standalone repo).
- Does not replace the Kubernetes scheduler.
- No Windows node support.
- No gRPC API (HTTP is sufficient).
- No UI/dashboard.

```go
package v1alpha1

import (
    corev1 "k8s.io/api/core/v1"
    metav1 "k8s.io/apimachinery/pkg/apis/meta/v1"
)

// +kubebuilder:object:root=true
// +kubebuilder:subresource:status
// +kubebuilder:resource:scope=Namespaced,shortName=swp
// +kubebuilder:printcolumn:name="RuntimeClass",type=string,JSONPath=`.spec.runtimeClassName`
// +kubebuilder:printcolumn:name="MinIdle",type=integer,JSONPath=`.spec.minIdleCount`
// +kubebuilder:printcolumn:name="Idle",type=integer,JSONPath=`.status.idleCount`
// +kubebuilder:printcolumn:name="Assigned",type=integer,JSONPath=`.status.assignedCount`
// +kubebuilder:printcolumn:name="Ready",type=string,JSONPath=`.status.conditions[?(@.type=="PoolReady")].status`
// +kubebuilder:printcolumn:name="Age",type=date,JSONPath=`.metadata.creationTimestamp`

type SandboxWarmPool struct {
    metav1.TypeMeta   `json:",inline"`
    metav1.ObjectMeta `json:"metadata,omitempty"`

    Spec   SandboxWarmPoolSpec   `json:"spec"`
    Status SandboxWarmPoolStatus `json:"status,omitempty"`
}

type SandboxWarmPoolSpec struct {
    // RuntimeClassName references a Kubernetes RuntimeClass configured for
    // gVisor (handler: runsc) or Kata Containers (handler: kata-qemu / kata-clh).
    // +kubebuilder:validation:Required
    RuntimeClassName string `json:"runtimeClassName"`

    // MinIdleCount is the target number of warm idle pods to maintain.
    // +kubebuilder:validation:Minimum=0
    // +kubebuilder:default=2
    MinIdleCount int32 `json:"minIdleCount"`

    // MaxPoolSize is the hard cap on total pods across all states.
    // +kubebuilder:validation:Minimum=1
    // +kubebuilder:default=10
    MaxPoolSize int32 `json:"maxPoolSize"`

    // PodTemplate defines the spec for each sandbox pod.
    // RuntimeClassName is injected automatically; do not set it here.
    // +kubebuilder:validation:Required
    PodTemplate corev1.PodTemplateSpec `json:"podTemplate"`

    // RecyclePolicy controls what happens when a pod is un-assigned.
    // Delete: terminate the pod and create a fresh one.
    // Reuse: attempt an in-place reset, return to Idle if successful.
    // +kubebuilder:validation:Enum=Delete;Reuse
    // +kubebuilder:default=Delete
    RecyclePolicy RecyclePolicy `json:"recyclePolicy,omitempty"`

    // IdleTimeoutSeconds: idle pods older than this are garbage-collected
    // when idleCount > minIdleCount. 0 = never expire.
    // +kubebuilder:default=300
    IdleTimeoutSeconds int64 `json:"idleTimeoutSeconds,omitempty"`
}

// RecyclePolicy determines pod fate after un-assignment.
// +kubebuilder:validation:Enum=Delete;Reuse
type RecyclePolicy string

const (
    RecyclePolicyDelete RecyclePolicy = "Delete"
    RecyclePolicyReuse  RecyclePolicy = "Reuse"
)

type SandboxWarmPoolStatus struct {
    // IdleCount is the number of pods in Idle state.
    IdleCount int32 `json:"idleCount"`
    // AssignedCount is the number of pods currently claimed by callers.
    AssignedCount int32 `json:"assignedCount"`
    // RecyclingCount is the number of pods being recycled.
    RecyclingCount int32 `json:"recyclingCount"`
    // PendingCount is the number of pods still initializing.
    PendingCount int32 `json:"pendingCount"`
    // TotalCount is the sum across all states.
    TotalCount int32 `json:"totalCount"`

    // Conditions reflects operational state.
    // +listType=map
    // +listMapKey=type
    Conditions []metav1.Condition `json:"conditions,omitempty"`

    // LastScaleTime is when the controller last created or deleted a pod.
    LastScaleTime *metav1.Time `json:"lastScaleTime,omitempty"`
}

// +kubebuilder:object:root=true
type SandboxWarmPoolList struct {
    metav1.TypeMeta `json:",inline"`
    metav1.ListMeta `json:"metadata,omitempty"`
    Items           []SandboxWarmPool `json:"items"`
}

func init() {
    SchemeBuilder.Register(&SandboxWarmPool{}, &SandboxWarmPoolList{})
}
```

### 6.3 Status Condition Types

| Condition Type | True When |
|---|---|
| `PoolReady` | `idleCount >= minIdleCount` |
| `PoolDegraded` | Pods are failing to reach Running (CrashLoopBackOff, ImagePullBackOff) |
| `PoolAtCapacity` | `totalCount >= maxPoolSize` |

## 7. Pod Lifecycle State Machine

Each pool pod carries the label `sandbox.koordinator.sh/pool-state`. The controller reads and transitions this label.

### 7.1 States

| State | Label Value | Description |
|---|---|---|
| Pending | `pending` | Pod created, waiting to become Running + Ready |
| Idle | `idle` | Pod is warm and ready to be claimed |
| Assigned | `assigned` | Pod claimed by a caller; carries assignment metadata |
| Recycling | `recycling` | Pod is being reset (Reuse) or about to be deleted (Delete) |
| Terminating | `terminating` | Delete has been issued; waiting for pod to disappear |

### 7.2 Transition Diagram

```
                    ┌─────────────────────────────────────┐
                    │           CREATE POD                │
                    └──────────────┬──────────────────────┘
                                   │
                                   ▼
                               Pending
                              /        \
                 Pod Running+Ready    Pod Failed
                        │                  │
                        ▼                  ▼
                        Idle         Terminating ──► (deleted; scaleUp creates replacement)
                          │
                  /assign HTTP call
                          │
                          ▼
                       Assigned
                          │
                  /unassign HTTP call
                          │
              ┌───────────┴───────────┐
         policy=Delete           policy=Reuse
              │                       │
              ▼                       ▼
        Terminating              Recycling
              │                       │
     (deleted; scaleUp         reset OK?
      creates replacement)    /         \
                            Yes          No
                             │            │
                             ▼            ▼
                            Idle     Terminating
```

### 7.3 Pod Labels & Annotations

Every pool pod carries:

```yaml
labels:
  sandbox.koordinator.sh/pool-name:   "<SandboxWarmPool.Name>"
  sandbox.koordinator.sh/pool-ns:     "<SandboxWarmPool.Namespace>"

annotations:
  sandbox.koordinator.sh/created-at:   "<RFC3339>"

  sandbox.koordinator.sh/assigned-to:  ""             # callerID, set on assignment
  sandbox.koordinator.sh/assigned-at:  ""             # RFC3339, set on assignment
```

### 8.2 Reconcile Loop (implement fully in code)

```
func Reconcile(ctx, req):
    1. Fetch SandboxWarmPool CR → if NotFound, return OK (already deleted)
    2. Handle deletion:
       - if DeletionTimestamp is set → delete all pool pods → remove finalizer → return
    3. Ensure finalizer "sandbox.koordinator.sh/pool-protection" is present
    4. List all pods: label pool-name=<name>, namespace=<namespace>
    5. syncPodStates(pool, pods):
       - For each pod, read pool-state label
       - Apply transitions per state machine (Section 7.2)
       - Patch state label via client.MergeFrom; retry on Conflict ≤3 times
       - Accumulate counts: idle, pending, assigned, recycling, terminating
    6. scaleUp(pool, counts):
       - need = max(0, minIdleCount - (idle + pending))
       - room = max(0, maxPoolSize - total)
       - toCreate = min(need, room)
       - for i in 0..toCreate: Create(buildSandboxPod(pool))
    7. expireIdlePods(pool, pods, counts):
       - skip if idleTimeoutSeconds == 0
       - for idle pods where time.Since(idle-since) > timeout AND idle > minIdleCount:
           Delete(pod); decrement idle
    8. updateStatus(pool, counts):
       - set status fields (idleCount, assignedCount, recyclingCount, pendingCount, totalCount)
       - set PoolReady condition: True if idleCount >= minIdleCount
       - patch status subresource
    9. Record Prometheus gauge for pool depth by state
   10. Return RequeueAfter: 15s
```

## 9. Pod Builder (`internal/controller/pod_builder.go`)

`buildSandboxPod(pool *SandboxWarmPool) *corev1.Pod` must:

- Deep-copy `pool.Spec.PodTemplate`
- Set `GenerateName: "swp-<pool.Name>-"`
- Set `Namespace: pool.Namespace`
- Inject labels: `pool-name`, `pool-ns`, `pool-state=pending`
- Inject annotations: `created-at=<now RFC3339>`, others empty string
- Set `OwnerReference` pointing to the `SandboxWarmPool` CR (enables garbage collection)
- Force `spec.runtimeClassName = &pool.Spec.RuntimeClassName`
- Force `spec.restartPolicy = Never` (controller owns lifecycle, not kubelet)
- If first container has no `readinessProbe`, inject a default:
  ```yaml
  readinessProbe:
    exec:
      command: ["true"]
    initialDelaySeconds: 2
    periodSeconds: 2
  ```

## 10. State Transition Helper (`internal/controller/state_machine.go`)

```go
// transitionState patches the pool-state label and relevant annotations on a pod.
// Uses MergeFrom patch with optimistic concurrency; retries up to 3x on Conflict.
func (r *SandboxWarmPoolReconciler) transitionState(ctx context.Context, pod *corev1.Pod, newState string) error {
    for attempt := 0; attempt < 3; attempt++ {
        patch := client.MergeFrom(pod.DeepCopy())
        pod.Labels[poolStateLabel] = newState
        switch newState {
        case stateIdle:
            pod.Annotations[idleSinceAnnot]  = metav1.Now().Format(time.RFC3339)
            pod.Annotations[assignedToAnnot] = ""
            pod.Annotations[assignedAtAnnot] = ""
        }
        err := r.Patch(ctx, pod, patch)
        if err == nil {
            return nil
        }
        if !errors.IsConflict(err) {
            return err
        }
        _ = r.Get(ctx, client.ObjectKeyFromObject(pod), pod) // re-fetch before retry
    }
    return fmt.Errorf("transitionState: exceeded retry limit on conflict")
}
```

## 11. HTTP Assignment Server (`internal/controller/assignment_server.go`)

Runs on port `8081` (configurable via `--assignment-server-addr`). Uses stdlib `net/http`. Receives `client.Client` injected at startup.

### 11.1 Concurrency Safety

Maintain a `sync.Map` keyed by `"namespace/poolName"`, each value a `*sync.Mutex`. Lock the pool mutex for the full duration of `/assign` to prevent double-assignment at the server level. Kubernetes-side patch optimistic concurrency is a second independent safety layer.

### 11.2 Endpoints

#### `POST /assign`

Claims one idle pod atomically.

**Request:**
```json
{
  "namespace": "default",
  "poolName":  "gvisor-pool",
  "callerID":  "job-runner-abc123"
}
```

**Logic:**
```
1. Lock pool mutex
2. List pods: pool-name=<poolName>, pool-state=idle, namespace=<namespace>
3. If empty → unlock → 503 { "error": "no idle pods", "idleCount": 0, "pendingCount": N }
4. Pick pod with oldest idle-since annotation (FIFO order)
5. Patch: pool-state=assigned, assigned-to=<callerID>, assigned-at=<now>
6. Unlock
7. 200 → { podName, podNamespace, podIP, nodeName, assignedAt }
```

**Response 200:**
```json
{
  "podName":      "swp-gvisor-pool-a3f92b1d",
  "podNamespace": "default",
  "podIP":        "10.244.1.45",
  "nodeName":     "node-01",
  "assignedAt":   "2025-08-01T10:30:00Z"
}
```

**Response 503:**
```json
{ "error": "no idle pods available", "idleCount": 0, "pendingCount": 2 }
```

---

#### `POST /unassign`

Returns a pod to the recycle pipeline.

**Request:**
```json
{ "podName": "swp-gvisor-pool-a3f92b1d", "podNamespace": "default" }
```

**Logic:**
```
1. Fetch pod; if pool-state != assigned → 400
2. If RecyclePolicy == Reuse: patch pool-state=recycling
   If RecyclePolicy == Delete: patch pool-state=terminating then Delete(pod)
3. Controller reconcile loop handles the rest
```

**Response 200:**
```json
{ "status": "recycling", "recyclePolicy": "Delete" }
```

## 13. Main Entrypoint (`cmd/main.go`)

```go
package main

import (
    "flag"
    "os"

    corev1 "k8s.io/api/core/v1"
    "k8s.io/apimachinery/pkg/runtime"
    utilruntime "k8s.io/apimachinery/pkg/util/runtime"
    clientgoscheme "k8s.io/client-go/kubernetes/scheme"
    ctrl "sigs.k8s.io/controller-runtime"
    "sigs.k8s.io/controller-runtime/pkg/healthz"
    "sigs.k8s.io/controller-runtime/pkg/log/zap"

    sandboxv1alpha1 "github.com/<you>/sandbox-warm-pool-controller/api/v1alpha1"
    "github.com/<you>/sandbox-warm-pool-controller/internal/controller"
)

var scheme = runtime.NewScheme()

func init() {
    utilruntime.Must(clientgoscheme.AddToScheme(scheme))
    utilruntime.Must(sandboxv1alpha1.AddToScheme(scheme))
    _ = corev1.AddToScheme(scheme)
}

func main() {
    var metricsAddr, assignmentAddr, probeAddr string
    var leaderElect bool

    flag.StringVar(&metricsAddr, "metrics-bind-address", ":8080", "")
    flag.StringVar(&assignmentAddr, "assignment-server-addr", ":8081", "")
    flag.StringVar(&probeAddr, "health-probe-bind-address", ":8082", "")
    flag.BoolVar(&leaderElect, "leader-elect", false, "")
    opts := zap.Options{Development: true}
    opts.BindFlags(flag.CommandLine)
    flag.Parse()

    ctrl.SetLogger(zap.New(zap.UseFlagOptions(&opts)))

    mgr, err := ctrl.NewManager(ctrl.GetConfigOrDie(), ctrl.Options{
        Scheme:                 scheme,
        MetricsBindAddress:     metricsAddr,
        HealthProbeBindAddress: probeAddr,
        LeaderElection:         leaderElect,
        LeaderElectionID:       "sandbox-warm-pool-controller.koordinator.sh",
    })
    if err != nil { ctrl.Log.Error(err, "unable to create manager"); os.Exit(1) }

    if err = (&controller.SandboxWarmPoolReconciler{
        Client: mgr.GetClient(),
        Scheme: mgr.GetScheme(),
    }).SetupWithManager(mgr); err != nil {
        ctrl.Log.Error(err, "unable to setup controller"); os.Exit(1)
    }

    srv := controller.NewAssignmentServer(mgr.GetClient(), assignmentAddr)
    go srv.Start()

    _ = mgr.AddHealthzCheck("healthz", healthz.Ping)
    _ = mgr.AddReadyzCheck("readyz", healthz.Ping)

    ctrl.Log.Info("starting sandbox warm pool controller")
    if err := mgr.Start(ctrl.SetupSignalHandler()); err != nil {
        ctrl.Log.Error(err, "problem running manager"); os.Exit(1)
    }
}
```

## 18. Testing Requirements

All tests in `internal/controller/`. Use `envtest` — real API server, no mocks of the Kubernetes client.

### Suite setup (`suite_test.go`)

Standard kubebuilder `envtest` pattern: start local API server, register the scheme, install CRD, start the reconciler manager in the test process, defer teardown.

### Controller Tests (`controller_test.go`)

| Test | Validates |
|---|---|
| `TestScaleUp_CreatesMinIdlePods` | CR minIdleCount=3, 0 pods → reconcile creates 3 Pending pods |
| `TestPendingToIdle_OnPodReady` | Pod Running+Ready → label transitions to `idle` |
| `TestRespectsMaxPoolSize` | minIdleCount=10, maxPoolSize=5 → only 5 pods created |
| `TestFailedPodDeleted` | Pod enters Failed phase → deleted; replacement created |
| `TestIdleTimeout_ExpiresExcessPods` | idle > minIdleCount + elapsed > timeout → pod deleted |
| `TestFinalizer_AddedOnCreate` | New CR → finalizer present |
| `TestDeletion_CleansAllPods` | CR with DeletionTimestamp → all pods deleted, finalizer removed |
| `TestStatusUpdated_Correctly` | After reconcile → status counts match actual pod labels |
| `TestRecyclePolicy_Delete` | Un-assign pod → pod deleted, fresh pending pod created |
| `TestRecyclePolicy_Reuse` | Un-assign pod → pod enters Recycling → transitions to Idle |

### Assignment Server Tests

| Test | Validates |
|---|---|
| `TestAssign_Success` | 1 idle pod → 200, pod transitions to assigned |
| `TestAssign_NoIdlePods` | 0 idle pods → 503 with counts |
| `TestAssign_ConcurrentRequests` | 5 concurrent assign calls, 1 idle pod → exactly 1 success |
| `TestUnassign_Success` | Assigned pod → 200, transitions to recycling |
| `TestStatus_ReturnsCorrectCounts` | Mixed-state pool → correct JSON response |
| `TestHealthz` | GET /healthz → 200 |

### Metrics Tests

Verify after operations:
- `swpc_pool_pods` gauge reflects correct counts per state
- `swpc_pods_created_total` increments on pod creation
- `swpc_assignment_duration_seconds` is observed on every `/assign` call
